【概述】
本项目所有商业化入口均通过 js/config.js 和 js/app.js 中的开关控制。
前期可一键隐藏所有付费元素，纯免费试水；后期一键开启变现。
所有价格、次数、文案均为可配置项，不写死。

二、开启 / 隐藏方法
================================================================

【全局开关（一键隐藏所有商业化模块）】
打开 js/config.js，找到 commercial.enabled：
  true  → 显示所有付费入口
  false → 隐藏所有付费入口（纯净免费工具模式）

【细粒度开关（在 commercial.enabled = true 时生效）】
各子开关设为 true/false 即可单独控制：

  commercial.membershipPage = false;   // 隐藏会员中心页
  commercial.upgradePrompts = false;   // 隐藏结果区升级提示
  commercial.exportLock = false;       // 取消导出按钮加锁
  commercial.featureLock = false;      // 取消高阶功能置灰
  commercial.quotaSystem = false;      // 取消免费次数限制
  commercial.promoBanner = false;      // 隐藏首页促销 banner
  commercial.newUserGift = false;      // 关闭新人福利弹窗
  commercial.privateDomain = false;    // 隐藏私域引流区
  commercial.navMemberBtn = false;     // 隐藏导航栏会员按钮

【隐藏会员导航项】
打开 index.html，找到 id="nav-membership" 的元素，
添加 style="display:none" 或删除该行。

三、可配置项说明（js/config.js）
================================================================

1. 免费次数
   quota.freeDailyLimit: 3       // 免费用户每日次数
   quota.resetHour: 0            // 每日重置时间（默认凌晨0点）

2. 会员定价
   membership.plans[1].price: 19     // 月卡价格
   membership.plans[2].price: 199    // 年卡价格
   （修改 priceLabel 字段同步更新页面显示文案）

3. 功能权限
   functions[].tier: 'free' | 'paid'
   - 设为 'free' 则该功能对免费用户开放
   - 设为 'paid' 则该功能需要会员

4. 文案配置
   texts 对象包含所有提示文案，可按需修改：
   - upgradeModalTitle / upgradeModalBody：升级弹窗文案
   - quotaExhaustedTitle / quotaExhaustedBody：额度用尽弹窗（{used}/{limit} 为占位符）
   - exportLockedTip：导出按钮锁定提示
   - resultUpgradeTip：结果区升级引导文案
   - modelBadge*：模型标识文字

5. 活动配置
   promo.banner.text / link / bgClass：首页 banner 文案/链接/样式
   promo.newUserGift.*：新人福利弹窗所有文案
   promo.privateDomain.*：私域引流区文案

6. 品牌信息
   brand.name / slogan / shortDesc：产品名称和简介

四、后续对接修改点位
================================================================

【对接 DeepSeek API】
1. 修改 js/config.js：
   api.endpoint = 'https://your-deepseek-api-endpoint/chat/completions';
   api.mockEnabled = false;

2. 重写 js/api.js 中的 callDeepSeek 函数：
   - 将 fetch 请求改为真实的 DeepSeek API 调用格式
   - 根据 DeepSeek API 文档调整请求头、请求体格式
   - 解析真实返回数据并映射到现有 data 结构

3. API Key 管理：
   - 建议通过后端代理转发请求（避免前端暴露 API Key）
   - 或使用环境变量 / Vercel Edge Functions 等方式保护 Key

【对接支付系统】
1. 修改 js/app.js → confirmPayment() 函数：
   - 当前为 mock 支付，直接升级会员
   - 替换为真实的支付 SDK 调用（微信支付 JSAPI / 支付宝等）
   - 支付成功后回调中调用 Auth.upgradeTo()

2. 支付结果验证：
   - 建议通过后端接口验证支付结果
   - 前端仅做展示层更新，实际会员状态由后端确认

【对接登录系统】
1. 修改 js/auth.js → login() 函数：
   - 当前为 mock 登录，读取本地存储
   - 替换为真实的 OAuth / 手机号 / 邮箱登录流程
   - 登录成功后存储 token 到 localStorage

2. 导航栏登录按钮：
   - 修改 index.html 中「登录/注册」按钮的 onclick
   - 跳转到登录页或弹出登录弹窗

3. 权限校验：
   - 在 api.js 的 Authorization header 中携带真实 token
   - 建议后端做权限校验，前端仅做 UI 层控制

【字数限制对接】
在 js/app.js → _handleToolSubmit 函数中：
- 当前未校验文本长度
- 可根据 userTier 添加字数校验逻辑
- 免费版上限 5000 字，月卡 20000 字，年卡 50000 字

五、部署说明
================================================================

本项目为纯静态文件，无需后端即可部署：

1. Vercel：
   - 将整个项目文件夹拖入 Vercel 控制台
   - 或通过 CLI：vercel --prod

2. Netlify：
   - 将项目文件夹拖入 Netlify 部署区
   - Build settings 无需配置（纯静态）

3. 阿里云 OSS 静态托管：
   - 上传所有文件到 OSS bucket
   - 设置 index.html 为默认首页

4. GitHub Pages：
   - 推送项目到 GitHub
   - Settings → Pages → 选择分支 → Save

【注意事项】
- 确保所有资源路径为相对路径（当前已配置）
- Tailwind CSS 通过 CDN 加载，需要网络连接
- 如需离线使用，可将 Tailwind CSS 编译为独立文件
